# 产品命名建议

> 基于「自然语言 → 结构化操作 → Excel 可复现」的核心能力，为产品提供替代「LLM Excel」的新名称方案。

---

## 一、产品能力摘要

- **核心**：用自然语言描述 Excel 处理需求，由 AI 生成可执行的结构化操作，输出预览、公式与可下载的 Excel 文件
- **流程**：load → generate → validate → execute → complete（对话式、可多轮修正）
- **特色**：100% 与 Excel 公式可复现、支持多文件/多 Sheet、流式反馈
- **品牌意象**：AI 智能 + 数据/表格（绿 + 蓝）

---

## 二、命名方向与候选

### 1. 对话 / 语言 + 表格（突出「说人话办表格事」）

| 名称 | 含义 / 联想 | 特点 |
|------|-------------|------|
| **SheetTalk** | Sheet + Talk，和表格对话 | 好记、国际化、产品感强 |
| **GridSpeak** | 网格 + 说话 | 偏技术感，有辨识度 |
| **SaySheet** | Say + Sheet，说出你的表格 | 简短、动词感 |
| **TableChat** | 表格聊天 | 直白，略泛 |

### 2. 智能 / 洞察 + 数据（突出 AI 与数据）

| 名称 | 含义 / 联想 | 特点 |
|------|-------------|------|
| **SheetMind** | 表格大脑 | 强调「会思考的表格」 |
| **DataLingua** | Data + Lingua（语言） | 有气质，偏专业/数据产品 |
| **GridMind** | 网格思维 | 简洁、偏技术品牌 |
| **TableLens** | 表格透镜 | 强调洞察与聚焦 |

### 3. 动作 / 流程感（突出「一句话搞定表格」）

| 名称 | 含义 / 联想 | 特点 |
|------|-------------|------|
| **FormulaFlow** | 公式流 | 对应 load→generate→validate→execute 的流水线 |
| **Cellwise** | Cell + Wise，单元格智慧 | 简短、好记、偏工具感 |
| **Gridify** | 把需求「网格化」 | 动词感、现代感 |
| **Scribe** | 书记员 / 抄写员 | 帮用户「写」表格，有故事感 |

### 4. 经典词 / 拉丁感（偏品牌、不直白）

| 名称 | 含义 / 联想 | 特点 |
|------|-------------|------|
| **Tabula** | 拉丁语「板、表」 | 短、好读、有文化感，需查商标 |
| **Lattice** | 格子 / 晶格 | 结构感强，可能已有同名产品 |
| **Nota** | 拉丁「记号、笔记」 | 极简，易记 |

### 5. 中文友好 / 可中英并用

| 名称 | 含义 / 联想 | 特点 |
|------|-------------|------|
| **表言** (TableSpeak) | 用语言驱动表格 | 中文顺口，可作 Slogan |
| **格语** (GridTalk) | 格子里的话语 | 简短，有记忆点 |

## 三、Top 3 推荐

1. **SheetTalk**  
   - 直接传达「和表格对话」的产品形态，好记、适合做主品牌名，Logo/域名都容易延展。

2. **DataLingua**  
   - 突出「数据的语言」—— 自然语言 ↔ 表格操作，适合偏专业/数据分析场景，名字有辨识度。

3. **FormulaFlow**  
   - 强调从需求到公式的「流水线」与可复现性，对技术/专业用户有吸引力，也便于和「Pipeline」概念绑定。

## 四、当前倾向：DataLingua / FormulaFlow

在 **DataLingua** 与 **FormulaFlow** 之间二选一或并用时，可结合后续规划做决定。

## 五、后续扩展：数据分析

若产品会**增加数据分析能力**（如统计、可视化、洞察建议等），两个名称的延展性略有不同：

| 名称 | 与「数据分析」的契合度 | 说明 |
|------|------------------------|------|
| **DataLingua** | 高 | 「数据的语言」天然涵盖「用自然语言做分析、问数据、看洞察」—— 分析即另一种与数据对话的方式，品牌不必改口。 |
| **FormulaFlow** | 中 | 强在「公式流水线」与执行；分析功能可理解为 Pipeline 上的一类步骤，但名字本身更偏「转换/执行」，分析感需靠 Slogan 或副标题补足。 |

**建议**：若数据分析是明确的产品方向，**DataLingua** 更利于长期统一品牌（数据处理 + 数据分析都在「数据的语言」里）；若短期内仍以「公式生成与执行」为主、分析为补充，**FormulaFlow** 也完全可用，后续可用副标题如「智能表格与数据分析」做延展。

## 六、非英语命名（拉丁 / 希腊 / 其他）

用非英语词做品牌名，气质更独特、不易撞名，且多数短而好读。以下按语种分组，含义均与「表、格、数据、书写、流」相关。

### 拉丁语 (Latin)

| 名称 | 含义 | 读音提示 | 说明 |
|------|------|----------|------|
| **Tabula** | 板、表、书写板 | 塔布拉 | 直接对应「表」，tabula rasa = 白板，文化感强 |
| **Cella** | 小格、小室、单元 | 切拉 | 与 spreadsheet「单元格」呼应 |
| **Tabella** | 小表、小板、牌 | 塔贝拉 | Tabula 的指小形式，更柔和 |
| **Nota** | 记号、笔记、注解 | 诺塔 | 极简，易记，偏「记录」 |
| **Folium** | 叶、张（纸/表） | 佛利乌姆 | 一「张」表、一页，偏书面感 |
| **Scheda** | 片、表、卡片、表单 | 斯凯达 | 意大利语常用「表/表单」，好读 |
| **Charta** | 纸、文书、图表 | 卡尔塔 | chart 词源，偏「图表/文档」 |
| **Scriptor** | 书写者、书记员 | 斯克里普托 | 谁在「写」表格，有故事感 |
| **Ratio** | 计算、账目、理由 | 拉提奥 | 拉丁本义含「计算」，英文 ratio 已泛化 |
| **Tessera** | 小方块、令牌、格片 | 泰塞拉 | 网格/格子的联想 |

### 希腊语 (Greek)

| 名称 | 含义 | 读音提示 | 说明 |
|------|------|----------|------|
| **Pinax** | 板、表、牌、画板 | 皮纳克斯 | πίναξ，强「表格/板」联想 |
| **Pinakas** | 表、板 | 皮纳卡斯 | πίνακας，Pinax 的另一种形式 |
| **Lexis** | 词、表达、用语 | 莱克西斯 | λέξις，与「自然语言」契合 |
| **Taxis** | 顺序、排列、结构 | 塔克西斯 | τάξις， taxonomy / syntax 词根 |
| **Gramma** | 所写之物、字母、文书 | 格拉玛 | γράμμα，grammar 词根，偏「书写」 |
| **Stichos** | 行、列、句 | 斯提霍斯 | στίχος，与表格「行」对应 |
| **Metron** | 度量、尺度 | 梅特隆 | μέτρον，metric 词根，偏「计量/分析」 |
| **Schema** | 形式、图式、结构 | 斯凯玛 | σχῆμα，schema 已进入英文，偏「结构」 |

### 其他语种

| 名称 | 语种 | 含义 | 读音提示 | 说明 |
|------|------|------|----------|------|
| **Tavola** | 意大利语 | 桌子、表 | 塔沃拉 | 来自拉丁 tabula，发音柔和 |
| **Jadwal** | 阿拉伯语 | 表、日程表 | 贾德瓦尔 | جدول，表/日程的常用词 |
| **Nagare** | 日语 | 流 | 纳嘎雷 | 流（ながれ），与「公式流」概念一致 |
| **Luach** | 希伯来语 | 表、板、历 | 卢阿赫 | לוח，短而有力 |
| **Sankhya** | 梵语 | 数、计算 | 桑基亚 | सङ्ख्या，与「计算/数据」相关 |
| **Patrika** | 梵语/印地语 | 文档、页、报 | 帕特里卡 | पत्रिका，偏「文档/表」 |

### 非英语命名小结

- **最贴近「表/格」**：Tabula、Pinax、Cella、Tabella、Jadwal、Tavola  
- **偏「书写/记录」**：Nota、Scriptor、Charta、Gramma  
- **偏「结构/顺序」**：Taxis、Schema、Stichos  
- **偏「流/计算」**：Ratio、Nagare、Sankhya、Metron  
- **简短好记**：Nota、Cella、Lexis、Pinax、Luach  

确定前建议查一下商标与域名（尤其是 **Tabula**、**Nota**、**Pinax**、**Schema** 等常见词）。

## 七、小众语言与创造型命名

围绕产品三大能力做命名联想：

1. **自然语言 → LLM → 公式 → 执行**（语言入、公式出、可执行）
2. **思路解读**（分析可见、步骤可读、推理透明）
3. **原子化快捷复现**（小步可验证、操作可重放、结果可复现）

以下从小众/少数语言中选词，并按「语义」分组；**创造型组合**均含「表/格」词根，保证表格意味。先列「表/格」相关词，再作组合。

| **Argitaula** | Argi（清晰）+ Taula（表） | 清晰之表 | 思路解读 + 表格，巴斯克/加泰风 |

## 八、使用建议

- 若希望**大众化、好记**：优先 **SheetTalk**。
- 若希望**偏数据/分析品牌，且规划了数据分析**：优先 **DataLingua**。
- 若希望**突出引擎与流程**：优先 **FormulaFlow**。
- 若希望**非英语、有文化感**：优先 **Tabula**，或从第六节拉丁/希腊/其他语种中挑选。
- 若希望**紧扣「自然语言→公式→执行 + 思路解读 + 原子复现」且带表格意味**：优先创造型 **Hitzula**、**Argitaula**、**Urratstaula**、**Sonatabel**、**Tabeltehe**、**Bidetaula**（见第七节，均含表/格词根）。
